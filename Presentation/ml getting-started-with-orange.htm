Getting Started With Orange

Introduction

Orange is an open source data mining tool with very strong data visualization capabilities. It allows you to use a GUI (Orange Canvas) to drag and drop modules and connect them to evaluate and test various machine learning algorithms on your data. This hands-on tutorial will go through setting up Orange and getting familiar with its GUI components. We do this by exploring a sample data set with some visualization widgets included with Orange. Also note that Orange has a very strong python scripting environment as well. Here is Orange's scripting tutorial.

Visit Orange's homepage and download the latest version. It comes pre-packaged with Python 2.7, NumPy, PyQt and various other libraries. Even if you have them on your system it is best to install everything it asks you to. Installation is pretty straightforward, you just install everything it asks you to. Also check out the Orange Add-ons. If you are into bioinformatics or Text Mining I highly suggest getting those add-ons too.

First Steps

Once installed, double click the Orange canvas to start it up. Click on the Data > File widget to add it to your canvas.

Then double click the file widget to configure it. In the Data File drop down, choose 'browse documentation data sets'. A pop-up will open up with all the data sets that Orange comes pre-packaged with. Here is some information about the Data Sets Orange comes bundled with. Eventually you will want to use your own data sets. It is very easy to use a spreadsheet or Python to shape your data into one of Orange's multiple input file formats. Back to the File Dialog, pick the file named iris.tab. This classic dataset contains the flower measurements of three different species of the iris plant. After you pick the file, in the Info section you will see that it has recognized your file and shows that it contains 150 examples with 4 attributes. It has also recognized that we are classifying them into three different discrete classes. Close the dialog for now. If you want read about the file module or any other module go to Orange's widget catalog.

Now click on the Data Table widget and under the Visualize Tab, Distributions, Attribute Statistics and Scatterplot widgets to add those widgets to your canvas. Hover over each of these and notice their input and output types. Do the same with the file widget. You will note that the output of the file widget corresponds to the input of all our other widgets. Orange works by connecting the channels of compatible widgets to build work flows that help you analyze the data. So lets connect the output of the File Widget to each of our other widgets.

Lets explore each of the widgets to see what our data looks like. Open up the Data Table widget and notice that Orange has helpfully drawn colored bars in each cell to help us visualize the value of every continuous variable.

The color of the lines represent the class of the examples which in this instance is the type of plant. This way you can click any of the columns to sort the data and quickly see if types of plants (or classes) have obvious boundaries. Next open up the Attribute Statistics widget. Notice that the C or D beside your attributes shows whether your attributes are continuous or discrete. Note that Orange does include Discretize and b.si/doc/widgets/_static/Data/Continuize.htm Continuize widgets in case you need them. They would be added between the file widget and any other widgets to shape your data as it passes through them. Next click on our class, iris, to notice that our data is evenly distributed in each of the three classes.

After that, click on sepal width and notice in the lower left that we have 150 total values out of which 23 are distinct.

Also note the mean and standard deviation printed in red, and the median. Orange has drawn a nice scale with our min and max values and is showing us where the majority of our data lies by showing us the 25th and 75th percentiles. We can see that the upper 25th percentile shows a lot more variation in its sepal width then the middle 50 percentile. This shows us that the data is not evenly distributed in terms of sepal width and that most of the data, the middle 50th percentile, shows very similar sepal width.

Open up the Distributions widget, select petal length as the variable and select all the classes.

Notice that all plants of the setosa species (blue color) have a petal length always below 2cms. Also notice that if a plant has petal length above 5.75cms we are sure it is virginica (green), but there are some virginica below that length too. So virginica and versicolor (red and green) aren't linearly separable like setosa (blue). Now open up the Scatterplot widget, set the X and Y attributes as petal length and sepal length respectively and notice that the blue mass is clearly separated from the red and green, again indicating that setosa is linearly separable whereas the other two aren't.

If we drew a vertical line say x = 2.1cm, we could separate the blue from the rest. But our data has four attributes and we can only see two of them on the Scatterplot. In many cases, our real life files will have lots and lots of attributes. To find the most interesting visualizations for these kind of files, we can press the VizRank Button and then select Start Evaluating Projections.

Then we can select the one with the highest score. More information about this algorithm can be found in Orange's page about the Scatterplot module.

Now we're talking

All of this is fine and good, Orange is making everything easier but we still don't see how Orange can really help. Well, the power of Orange comes from how it makes interactive data analysis possible. Lets test that. Add another Scatterplot and Distribution widget to the canvas. Connect our original Scatterplot to each of them.

When you do this you will see a 'Connect Signals' pop-up.

As there are multiple compatible outputs from our first Scatterplot that can feed into the second Scatterplot Orange is asking us to make a selection. Click on the gray line Orange has drawn to delete it. Then you can connect any other channels. Lets connect the Selected Data to Data like it was before. This ability to configure the signals between the modules will be useful later on. Now open up both the Scatterplots and put them side-by-side on your screen. Notice that the new Scatterplot is empty. Draw a box in the original Scatterplot in the area where the red and green dots overlap. You will see that the second Scatterplot is directly updated. Now we can better focus on only the data that isn't linearly separable.

Let's try to build a Decision Tree for only this data in the middle that isn't linearly separable. Click on Classify > Classification Tree and Classification Tree Graph. Connect the first Scatterplot to the Classification Tree and that Tree to the Classification Tree Graph.

Open up the original Scatterplot and the Classification Tree Graph and put them side by side. In the original Scatterplot click the red X mark in the Zoom/Select to clear all selections. Now draw a box again around the part of data that overlaps and see that Orange has automatically built a decision tree for us for just this data!

Try drawing a box in different regions of the Scatterplot and see that Orange builds a new decision tree every time. This is the power of interactive data analysis that Orange makes available. Next time we will look at the decision tree widgets and some of the other machine learning algorithms in Orange.

Downloads:

  • Orange schema: This is the final state of the schema that we built here. Load this into Orange by downloading the file and selecting File > Load Schema in Orange.
  • Iris data set: This is the input data set. Download the file and select it in the File Widget.
  • Assignment: Choose any data set that isn't included with Orange but which interests you, convert that data set to Orange's file format and then analyze it in a little bit more detail then we did here. The idea here is to get comfortable in analyzing data interactively using Orange's tools. Hand in a report with screenshots and analysis.
